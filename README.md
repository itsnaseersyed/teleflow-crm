# 📞 TeleFlow CRM (Callwise CRM)

A modern, high-performance Customer Relationship Management (CRM) platform purpose-built for telecalling teams, admission counselors, and sales representatives. TeleFlow CRM streamlines lead tracking, call logging, and follow-up scheduling into one beautifully simple, lightning-fast workspace.

## ✨ Key Features

*   **Role-Based Access Control (RBAC):** Strict security boundaries. Admins have a global view of the company data and can manage users, while Telecallers have a focused workspace containing only their assigned leads and tasks.
*   **Centralized Lead Management:** Track every lead's journey from initial contact to conversion.
*   **Call & Follow-up Tracking:** Log outcomes of every call and automate scheduled follow-ups so no lead slips through the cracks.
*   **Admin Dashboard:** Master control panel for creating and managing Telecaller accounts, viewing global analytics, and maintaining data hygiene.
*   **Modern "Glassmorphism" UI:** A stunning, highly responsive user interface built with modern design principles for maximum user engagement.

## 🛠️ Technology Stack

*   **Frontend Framework:** [React 18](https://react.dev/) powered by [TanStack Start](https://tanstack.com/router) for robust routing and Server-Side Rendering (SSR).
*   **State Management:** [TanStack Query](https://tanstack.com/query) for efficient, cached asynchronous data fetching.
*   **Styling:** [Tailwind CSS v4](https://tailwindcss.com/) paired with [shadcn/ui](https://ui.shadcn.com/) for beautiful, accessible components.
*   **Backend & Database:** [Firebase](https://firebase.google.com/) 
    *   **Firebase Authentication:** Secure, session-based user login.
    *   **Cloud Firestore:** Real-time NoSQL database protected by strict, role-based Security Rules.
*   **Icons:** [Lucide React](https://lucide.dev/)

## 🔒 Security Architecture
This project utilizes a custom dual-layer authentication setup:
1. **Public Signup Disabled:** To prevent unauthorized access, public account creation is disabled.
2. **Admin-Only Provisioning:** The Master Admin securely provisions all Telecaller accounts directly from the internal dashboard using a secondary Firebase App instance, ensuring strict control over who can access the CRM.
3. **Firestore Rules:** Database-level security guarantees that users can only read or write data they explicitly own or are assigned to.

## 🚀 Getting Started

### 1. Installation
Clone the repository and install the dependencies:
```bash
npm install
```

### 2. Environment Setup
Create a `.env.local` file in the root directory and add your Firebase configuration:
```env
VITE_FIREBASE_API_KEY="your_api_key"
VITE_FIREBASE_AUTH_DOMAIN="your_auth_domain"
VITE_FIREBASE_PROJECT_ID="your_project_id"
VITE_FIREBASE_STORAGE_BUCKET="your_storage_bucket"
VITE_FIREBASE_MESSAGING_SENDER_ID="your_sender_id"
VITE_FIREBASE_APP_ID="your_app_id"
```

### 3. Master Admin Initialization
Since the public signup page is disabled for security reasons, the master admin account must be created manually one time:
1. Go to your **Firebase Console** > **Authentication** > **Users**.
2. Click **Add User** and create an account with your desired admin email and password.
3. Copy the **User UID** generated by Firebase.
4. Go to **Firestore Database** and create a `users` collection (if it doesn't exist).
5. Create a document where the **Document ID** is exactly the UID you copied.
6. Add the following fields to the document:
   - `email` (string) = Your admin email
   - `role` (string) = `admin`
   - `fullName` (string) = `Master Admin`

### 4. Running Locally
Start the Vite development server:
```bash
npm run dev
```

Visit `http://localhost:3000` to access the application. Log in with your Master Admin credentials to begin creating Telecaller accounts.
